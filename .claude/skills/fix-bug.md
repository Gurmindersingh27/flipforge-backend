# Fix Bug
Before making any changes:
1. Restate the bug in plain English
2. Identify the exact file and code path causing the issue
3. Explain the root cause
4. Propose the smallest possible fix
5. List the files that will change
6. Show expected diff
7. Wait for approval
Rules:
- Do not refactor unrelated code
- Do not modify APIs unless required
- Do not introduce new abstractions
- Keep the fix minimal
